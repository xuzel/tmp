## ① ⚡ 30 秒速记卡（Core Terms）

- **Autonomous system** = 能 **无需人类干预** 执行任务的系统
- **AV（Autonomous Vehicles）** = 自动驾驶汽车
- **AWS（Autonomous Weapons Systems）** = 自主武器系统
- **Trolley Problem** = 电车难题（Topic 2 思想实验 → AV 实时版）
- **Moral Machine experiment** = MIT 实验揭示自动驾驶伦理的文化差异
- **Vulnerable road users** = 脆弱道路使用者（行人、骑行者）
- **Transition period（过渡期）** = 人类驾驶与自动驾驶混行阶段
- **Ethics-by-design** = 伦理即设计（类比 Privacy-by-Design）
- **Formal verification & validation（V&V）** = 形式化验证与验证
- **Shared responsibility** = 人类与 AI 系统共担责任
- **Harmonization of laws** = 国际法律协调
- **Distinction（区分原则）** = 国际人道法 IHL 核心：战斗员 vs. 非战斗员
- **Proportionality（比例原则）** = IHL 核心：军事行动与目标成比例
- **Human dignity（人类尊严）** = AWS 反对论据的核心概念
- **Lower the threshold for armed conflict** = AWS 降低战争门槛
- **Politically palatable warfare** = 政治上更可接受的战争（AWS 侵蚀反战动力）
- **Asymmetric warfare** = 非对称战争
- **Arms race** = 军备竞赛
- **Swarm warfare** = 蜂群战争（成千上万自主无人机协同）
- **Meaningful human control** = 有意义的人类控制（军控辩论核心术语）
- **Rules of engagement** = 交战规则
- **Fail-safe mechanisms** = 失效安全机制
- **Situational awareness** = 态势感知
- **Human-machine teaming** = 人机协同
- **Moral uncertainty（道德不确定性）** = 承认"不知道正确答案"的伦理立场
- **Ethical risk assessment** = 伦理风险评估
- **Product liability law** = 产品责任法（需重新评估以应对 AV）
- **IHL** = International Humanitarian Law（国际人道法）
- **CCW** = Convention on Certain Conventional Weapons（联合国常规武器公约）

## ③ ⚠️ 易混概念对比表

### A. AV（自动驾驶） vs. AWS（自主武器）
| | AV | AWS |
|---|---|---|
| 场景 | 民用道路 | 军事战场 |
| 频率 | 高频日常 | 低频高危 |
| 核心伦理 | **Trolley Problem** 实时版 | 生死决定权 |
| 关键原则 | 94% 人为失误消除 | Distinction + Proportionality |
| 监管重点 | Product liability | IHL + 国际军控 |

### B. Distinction vs. Proportionality（IHL 双原则）
| | Distinction（区分） | Proportionality（比例） |
|---|---|---|
| 含义 | 区分战斗员 vs. 非战斗员 | 军事行动与目标成比例 |
| 违反例 | 杀害平民 | 过度杀伤 |

### C. "Ethics-by-Design" vs. "Privacy-by-Design"
- 都是"从第一天嵌入"的理念
- **Ethics-by-Design** = 本讲（自主系统）
- **Privacy-by-Design** = Topic 4（隐私）

### D. Moral Agency vs. Shared Responsibility vs. Strict Liability
| | Moral Agency | Shared Responsibility | Strict Liability |
|---|---|---|---|
| 含义 | AI 能否为责任主体 | 人+AI 共担 | 不论过错即责 |
| 属于 | 哲学概念 | 法律难题 | 法律制度 |

### E. Meaningful Human Control 的 3 层
```
Intent Level（战略意图）
        ↓
Rules Level（交战规则）
        ↓
Execution Level（规则内自主执行）
```
**不是**要求"人类按每次扳机"

### F. V&V 缩写易错
- **Verification**：系统是否 **按规约** 构建？（"build right"）
- **Validation**：系统是否 **满足需求**？（"right build"）

### G. AWS 降门槛的"5 大"
1. Speed/Scale 改变
2. 更快升级
3. 军备竞赛
4. **Politically palatable**
5. Asymmetric warfare

## ⑥ 🕳️ 常见出题陷阱

1. **自主系统定义关键词是 "without human intervention"**——不是"全自主独立"。
2. **AV 核心数据：全球每年交通死亡 ≈ 130 万；94% 源于人为失误**——别记错数字。
3. **Trolley Problem 在本讲是"实时版"**（真实道路发生），不再只是思想实验。
4. **Moral Machine 实验展示的是"文化差异"**，不是"伦理统一方案"。
5. **Distinction 是"区分战斗员"，Proportionality 是"行动与目标成比例"**——两原则不要混。
6. **"Politically palatable" 指战争变得政治上可接受**，不是字面"可口"。
7. **Meaningful Human Control ≠ 每次按扳机**——是意图/规则/执行 3 层。
8. **Ethics-by-design 是工程理念**，与 Privacy-by-design 一脉相承（Topic 4）。
9. **V&V = Verification + Validation**，不是 Validation 的两次。
10. **Shared Responsibility 是人+AI 共担**，不是多 AI 间共担。
11. **Moral uncertainty 是伦理学前沿概念**——承认"不知道正确答案"。
12. **5 步伦理分析法的顺序**：Stakeholders → Consequences → Frameworks → Moral Uncertainty → Trade-offs，不要打乱。
13. **Stop Killer Robots 是反 AWS 运动**，不是支持方。
14. **4 大工程原则**：Ethics-by-Design / Domain-Specific / Interdisciplinary / V&V——别漏第 4 条。
15. **AWS "降低门槛"意指"更容易开战"**，不是"降低杀伤力"。
